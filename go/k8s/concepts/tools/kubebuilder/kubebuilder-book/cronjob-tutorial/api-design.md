＃设计一个API

在Kubernetes中，我们有一些设计API的规则。即全部
序列化的字段必须是camelCase，因此我们使用JSON结构标签
指定这个。我们也可以使用omitempty结构标记来标记
如果为空，则序列化中应省略该字段。

字段可以使用大多数原始类型。数字是例外：
为了实现API兼容性，我们接受三种数字形式：`int32`
和“ int64”代表整数，“ resource.Quantity”代表小数。
